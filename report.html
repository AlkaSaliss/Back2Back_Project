Back2Back testing of machine learning algorithms

ABOUBACAR ALKA Mahamadou Salissou, FORMAL Thibault, GIRARD Naomi, KEITA Seydou

Context & objectives

In this project, we aim to build a tool allowing us to easily realize Back2Back testing for machine learning algorithms.

Formally, Back to Back testing is performed to check the likeliness of operational results for a series of inputs on two or more similar components of different versions.

With the recent growing interest in Machine Learning in production systems, many frameworks/libraries have been developped. From the former (SAS, Weka etc.) to the new contenders (eg Tensorflow), Data Scientists have access to a wide range of tools, depending on objectives/needs (PoC, Distributed DataBases, Deep Learning etc.). As a result, many machine classical learning algorithms (Linear Regression, Decision Trees, Random Forests etc) are part of these frameworks.

It it then interesting to compare the behavior of these algorithms (which should behave the same, no matter their implementation). For instance, for the same data set, we would like to know if similar algorithms (let’s say a decision tree) lead to the same results (in terms of metrics, accuracy for instance). We could then compare their performance in term of computation time.

For this project, we will focus on three ML libraries:

  • Weka, is a suite of machine learning software written in Java, and has been in the game for a long time now.

  • Spark ML is Spark Machine Learning Library, allowing to design ML algorithms in a distributed environment.

  • renjin is an alternative interpreter for the R programming language designed to run in the JVM.

The main objective of this project is to build a tool allowing us to simply launch test procedures among different libraries, by abstracting the intern mechanisms of each library. For this purpose, we try to focus on software engineering aspects in order to propose a soft interface.

This project will serve as a basis for a larger project, which will include a web interface for back to back testing. Hence, we can imagine a scenario where a user load a dataset in a csv file, give some basic information about the data (the column to explain Y, if there is a header etc), a type of model (e.g. random forest), set certain hyperparameters according to the model (for instance number of trees in our case), and can then launch a comparison procedure, which will display in real time results like metrics or even better graphs.

General comments concerning the project

  • During the project, we consider only two kinds of machine learning algorithms: decision trees & random forests. The way we have structured the code allows to easily add other ml algorithms.
  • These two algorithms both treat regression and classification. Hence, we need two kinds of metrics in order to evaluate models.
